## 〇、会议确认的事实

- RLVR-World 奖励 = **MSE/MAE + LPIPS(vgg)**，两项、**等权 1:1**、**全解码后**、全图、无真正特征层（LPIPS 虽用 VGG 特征，但在解码图上算）。代码核实无误。
- 我们方案现在"东西很多很杂"：图像/特征、解码前/后、多分量、多粒度 → 导师认可"信息都有用，但**重要程度不一样**"。

---

## 一、导师要求**先钉死**的概念：解码前 vs 解码后（会上的混淆点）

**关键澄清：「特征层 vs 图像层」和「解码前 vs 解码后」是两条不同的轴，会上把它们混了。**

|  | **图像/像素级** | **深度特征级** |
|---|---|---|
| **解码后**（要先 decode 成图像） | MSE、MAE、PSNR、SSIM | LPIPS、DISTS、DINOv2(逐对)、FID/FD-DINOv2/CMMD(分布) |
| **解码前**（直接在 token/码上，无 decoder） | —— | **FSQ 码空间 RMS（本文 DoR 核心）**、phi 连续特征 |

**必须向导师明确的结论：**
1. **FID / FD-DINOv2 / LPIPS 都是「解码后」**——它们都要先有图像才能抽特征，**不是解码前**（会上我一度说成解码前，是错的）。
2. **唯一真正「解码前」的是 FSQ 码空间**（`indices_to_codes`，不过 decoder）。
3. 所以"加特征层"≠"加解码前"。我们的卖点是**解码前（码空间）**；FID/DINOv2 那些是**解码后的特征评测**（进 Evaluator，不进 reward，因为分布级给不了单候选打分）。

→ **待办**：reward 设计里把**解码后分量**和**解码前分量**分两块单独写清（导师明确要求"解码后单独写、解码前单独写"）。

## 二、第一步（奖励分量）：问题 + 动机

**动机（导师原话提炼）**：解码后指标直观、以图像为主、希望真实——**但它有误差（地板）**；所以前端（解码前）需要一个**更强的额外监督**，而且**前端可能起主导作用**。
→ 这就是"为什么加解码前码空间分量"的动机：解码后带地板，解码前≈零地板、信号更干净。

**这一步的产物**：一组跨"解码前/后 × 图像/特征 × 多粒度"的误差分量。

---

## 三、第二步（核心问题）：一堆异质误差**直接喂原始 GRPO，会出什么问题？**

> 这是导师最强调、要求"说明白"的一步。先讲问题，再讲我们的解决。

### 3.1 问题（清楚、简单地陈述）
把跨解码前后、图像/特征、多粒度的**异质误差等权合成一个奖励、直接喂原始 GRPO**，有四个问题：

1. **异质可靠度被抹平**：各分量地板高低不同（进展4 实测：LPIPS/SSIM 地板≥信号、MSE/code 地板<信号）。**等权 = 让高地板（噪声主导）分量和干净分量同等发言** → 噪声灌进组内排序 → GRPO 学噪声。（本文 thesis 在多分量下的放大版。）
2. **退化组**：某些样本整组信号 < 地板，其组内排序纯噪声；**原始 GRPO 照样从中算优势并更新** → 学噪声。
3. **GRPO 归一化本身有偏**（Dr.GRPO 指出）：除以组内 std 在噪声主导时引入偏置。
4. **重要程度随样本变**：固定等权无法适应——同一分量对某些样本有用、对另一些可能是**负面**（导师明确点了这条）。

### 3.2 解决方案（针对每个问题，简单/难都行，但要有道理）
- **分量级 → 治 1、4**：地板感知**加权** $w_m$（高地板分量降权）。
- **组级 → 治 2**：地板感知**过滤 / 动态采样**（丢弃 $\sigma_\star^2 \lesssim \sigma_\eta^2$ 的退化组）。
- **算法级 → 治 3**：**去偏 GRPO（Dr.GRPO）**。

> 导师提醒：单纯"简单分配权重 / 学习权重"**稍微弱**——所以要叠上**过滤（去偏/动态）**，并说清每招治哪个问题。

## 四、GRPO 变体：用哪个？我们的小改是什么？

会上确认有两类改进（文献已核，见 `lit_metrics_grpo.md`）：
- **Dr. GRPO（去偏）**：修 GRPO 归一化偏置 → 治问题 3。
- **DAPO（动态采样）**：训练时**过滤退化组**（组内奖励全同的组不更新）→ 治问题 2 的机制来源。

**我们的方案（第二创新点）**：不另造 GRPO，而是 **= Dr.GRPO（去偏）+ DAPO 式动态采样，并把过滤判据换成"地板感知"**：

$$
\text{保留组 } g \iff \mathrm{Var}_{k}\big(r^{\mathrm{code}}_{g}\big) \;\gtrsim\; \tau\,\phi^2
$$

即**用解码前 code 分量的组内方差是否超过地板阈值**来筛（导师说的"一行函数"）。先过滤、再交给（去偏的）GRPO。
**待定**：$\tau$ 怎么定（建议由进展4 的地板离线定）；过滤放在 GRPO 前还是与 advantage 一起。

## 五、$w_m$ 权重的来源（不能凭空，必须有链条）——回应我自己的疑问

完整链条（详见 `reward_design.md` 第六节）：
1. **推导**：把各分量视为真质量 $q$ 的带噪观测，最优合并权 = 逆方差/可靠度加权（Gauss–Markov / 维纳，标准结论），$w_m=\sigma_\star^2/(\sigma_\star^2+\phi_m^2)$ = 可靠度（信号占比）。**有出处。**
2. **测量**：$\phi_m$ 来自进展4 往返地板（实测）、$\sigma_\star^2$ 来自缓存的组内方差（离线估）。**输入非自由参数。**
3. **诚实弱环 + 验证**：各度量测的不是同一个 $q$ → 不宣称最优，须实验验证：
   - **V1**：floor-weighted reward vs 等权 vs 单分量（赢了才算有用）。
   - **V2**：权重温度 $w_m^\tau$ 扫描，看性能峰值是否落在地板推导权重附近。
   - **V3 兜底**：若连续式不灵，退到**硬门控**（丢 $\phi_m>\sigma_\star$ 的地板主导分量、其余等权）。
   - 纪律：若 V1/V2 不成立，**如实改等权/门控，不硬编赢的权重**。

## 六-补、地基：输入 GRPO 的到底是什么（必须先钉死）

**结论：输入 GRPO 的是「每个候选一个标量奖励 $R_i$」——所有分量算完、合成后的那一个总奖励。不是各分量、不是优势。**

数据流（RLVR、我们、标准 GRPO 都一样）：

```
1 个 transition → 采 G 个候选 {o_i}
  每候选: 算各分量 d_m(i) → 合成成 1 个标量 R_i     ← 喂进 GRPO 的就是它
GRPO 内部自己算:  A_i=(R_i−mean_G R)/(std_G R+ε) → 损失 −A_i·logπ(o_i)+KL
```

- **喂的是 reward，不是 advantage**；优势是 GRPO **自己**做组内归一化算的。
- **是"总奖励"(合成后的一个数)**，GRPO 看不到各分量。
- **RLVR 一样**：代码 `reward_tensor[i, last]=-(MSE+LPIPS)`，一候选一标量，放最后一个 token。

### 地基锁定（消除下午卡住的歧义）
1. **采用 (A) 纯逐候选奖励**：$R_i$ 用**固定离线尺度** $s_m$ 合成，**不在 reward 里做组内 z-score**。组内归一化**只由 GRPO 做一次**。→ 结构平行 RLVR、职责最干净。
2. **合成在前、归一化在 GRPO**：多分量/多空间/多粒度/地板加权 → 先塌成 $R_i$ 一个标量 → 再进（改进版）GRPO。
3. **"地板感知"分两处落**：① reward 里的逐候选权重 $w_m$（一阶处理）；② GRPO 的**组过滤**（主创新，治退化组）。**不塞进 reward 的组内归一化。**

## 六-补2、reward 与权重 $w_m$ 的最终口径（钉死）

**最终奖励（逐候选，一行）：**
$$
R_i=-\Big(w_{\mathrm{code}}\tfrac{\mathrm{RMS}(z_i-z')}{s_{\mathrm{code}}}+w_{\mathrm{recon}}\tfrac{\mathrm{MSE}(\hat s'_i,s')}{s_{\mathrm{recon}}}+w_{\mathrm{perc}}\tfrac{\mathrm{LPIPS}(\hat s'_i,s')}{s_{\mathrm{perc}}}\Big)
$$
- 解码前 1 项（code）+ 解码后 2 项（MSE、LPIPS），分块清楚。
- $s_m$=离线尺度（数据集 std，定值）；RLVR = 特例 $\{w_{\mathrm{code}}=0,\ w=s=1\}$。

**权重 $w_m$ 的来源（三段链，不凭空）：**
- **来源**：$w_m=\dfrac{\sigma_\star^2}{\sigma_\star^2+\phi_m^2}$ = 可靠度/信号占比（逆方差/维纳加权，估计理论标准结论）。
- **测量**：$\phi_m$=进展4 往返地板（实测）、$\sigma_\star^2$=缓存组内方差（离线估）。**无自由旋钮。**
- **定位（按导师"权重稍弱"的判断）**：$w_m$ 是**一阶处理**；真正的强项是 GRPO 的**地板感知组过滤**。
- **验证（必须，否则就是凭空）**：V1 floor-weighted vs 等权 vs 单分量；V2 权重温度扫描看峰值；V3 兜底=硬门控（丢 $\phi_m>\sigma_\star$ 的分量）。V1/V2 不过 → 如实退等权/门控。

> 一句话给导师：**进 GRPO 的永远是"合成后的一个标量奖励"（RLVR 也是）；我们把这个标量做成"地板感知加权的多空间和"，并在 GRPO 端加"地板感知组过滤"。权重有推导+测量+验证三段链，不硬编。**

## 六-补3、第二创新点钉死：用哪个 GRPO + 过滤判据

### 选型：以 **Dr. GRPO（去偏）为基** + **地板感知动态过滤**
**为什么是 Dr.GRPO 而非 DAPO（理论契合，非随便挑）：**
- 标准 GRPO 优势 $A_i=(R_i-\mathrm{mean})/\mathrm{std}$。当一组**地板主导**（$\sigma_\star^2$ 小、std 被地板噪声 $\sigma_\eta$ 灌满）时，**除以 std 会把这组的噪声排序放大到单位方差**，当成强梯度喂进去 —— 正是本文 thesis 的失效。
- **Dr.GRPO 的改动 = 去掉 /std**（$A_i=R_i-\mathrm{mean}$）。于是地板主导的组（spread 小）**自然只贡献很小的优势**，不再被放大。**它的"去偏"和我们的"去地板"是同一现象的两面。** → 取 Dr.GRPO 为基。
- DAPO 的价值是它的**动态采样机制**（训练时丢退化组）；我们**借这个机制、把判据换成地板感知**（下）。

### 过滤判据（导师要的"一行函数"）
**动机/问题**：σ_⋆ 极小的组里，连干净的 code 奖励都分不开候选，post-decode 分量纯地板噪声 → 该组优势无意义，更新它=加噪声。DAPO 原版只丢"奖励全同"的组；我们**用地板把它一般化**：

$$
\boxed{\ \text{保留组 } g \iff \mathrm{std}_{k}\big(\tilde r^{\mathrm{code}}_{g}\big) \;>\; \tau\,\tilde\sigma_\eta\ }
$$

- $\tilde r^{\mathrm{code}}$：尺度归一后的 **code 奖励**（解码前、≈零地板）→ 其组内 spread = 该组**真动态信号** $\sigma_\star$（用 floor-free 的 code 来量信号，最干净）。
- $\tilde\sigma_\eta$：地板噪声尺度，由**进展4 实测**标定（同一归一空间）。
- $\tau\!\approx\!1$：信号超过地板才保留；$\tau$ 由进展4 定默认 + 小扫描，**非拍脑袋**。

**一句话**：丢掉"真信号没超过地板"的组，其余交给去偏(no-/std)的 GRPO。

### 第二创新点 = 「Floor-aware GRPO」
= **Dr.GRPO 去偏（no /std，防地板组被放大）** + **地板门控动态采样（丢 $\sigma_\star\!\le\!\tau\sigma_\eta$ 的组）**。两招都直接对着"地板腐蚀组内排序"这个问题，且都是在已有改进(Dr.GRPO/DAPO)上的小改——符合"无伤大雅小改"。

> 退化保证：$\tau\!\to\!0$（不过滤）+ 保留 /std → 退回原始 GRPO。
