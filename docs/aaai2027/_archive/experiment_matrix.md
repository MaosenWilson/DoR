# DoR 实验矩阵（AAAI 2027）

> **依据**：`docs/DoR_AAAI2027_story_spine.md`（脊柱 + 已锁决定）。本文件把脊柱落成可执行的实验计划。
> **算力**：主力 RTX 5090 (32G, sm_120, HF generate + 自定义 GRPO)；H800 (80G) 备用。**deadline 7 月底**。
> **已锁决定**：符号 $\phi_{\text{tok}}$；机理=排序翻转概率 bound；真质量参照=DINOv2(held-out)+oracle tokenizer 交叉；下游闭环已砍，用 FVD+运动保真度+秩保持度预测图替代。

## 0. 实验设计三条控制（不可违反）

1. **同起点**：所有 RL 臂从同一 pre-RL base ckpt 出发（`thuml/rt1-world-model-single-step-base`），同 $G$、同步数、同超参，只换奖励度量空间。
2. **多 seed**：每个臂 ≥5 seed，报均值 ± std，看误差棒是否重叠。
3. **独立评测**：主结论只用训练目标之外的指标（LPIPS / PSNR / FVD / 运动保真度）；`code RMS` 等同源指标仅进附录。**reference（DINOv2）绝不当奖励臂**。

---

## 1. 奖励臂集合（空间扫描，全部 GT 锚定、喂同一 GRPO）

度量空间从"解码后"逐步移到"解码前"，这是全文主轴。tokenizer (`CNNFSQModel256`) 已现成提供 `encode_features`（连续预量化特征 φ）与 `indices_to_codes`（后量化码 z），**非像素臂全部自带，无需外部 VAE**。

| 臂 | 奖励度量 $D$ | 空间 | 地板 $\phi_{\text{tok}}$ | 角色 |
|---|---|---|---|---|
| **A0** | −LPIPS | 像素（解码后） | 高(~0.053) | RLVR-World 原版主基线 |
| **A1** | −MSE / PSNR | 像素 | 高 | 像素对照 |
| **A2** | SSIM | 像素 | 高 | 结构对照 |
| **A3** | **floor-校准像素**（−LPIPS 减/归一化 $\phi_{\text{tok}}$） | 像素 | 名义降低 | **最致命 baseline**，必须明显输给 code |
| **A4** | ToolRL 式多分量（PSNR+SSIM+LPIPS 融合） | 像素 | 高 | 证明不是"随便更好的像素奖励" |
| **A5** | −RMS(φ 预量化连续特征) | 解码前·连续 | 低 | 非像素低地板臂 |
| **A6** | **−RMS(z 码空间) = DoR** | 解码前·量化码 | ≈0 | **方法主臂** |

> A5 vs A6 顺手回答"量化到底帮还是伤奖励信号"，是一个干净的内生消融。

## 3. 独立评测指标

| 指标 | 空间/性质 | 用途 | 工具 |
|---|---|---|---|
| LPIPS / PSNR | 像素，独立于 code 目标 | 主结论 | 现有 `Metrics` |
| FVD | 分布级视频质量 | 主结论（见风险 R1：单步需短 rollout） | I3D 离线抽特征 |
| 运动保真度 | 光流一致性 sim(flow(ŝ′),flow(s′)) | 直接测动态，正中 thesis | RAFT（冻结）|
| 秩保持度 | 各奖励 vs DINOv2 参照的组内 Spearman/Kendall | 机理 + 选空间判据 + 预测性 | `probe_phi_dino` 扩展 |
| code RMS | 同源 | 仅附录 | 现有 |

## 5. 实验块、优先级与算力分工

| Block | 内容 | 产出 | 平台 | 优先级 |
|---|---|---|---|---|
| **B0 离线分析（先做，去风险）** | 缓存候选 + 地板刻画 + 秩保持度分析；**不训练即可验证 thesis** | Fig 1, Fig 2, Fig 3 雏形 | 5090 | ★★★ 最高 |
| **B1 核心空间扫描训练** | A0–A6 + C1，RT-1 single-step，步数从 40 拉到 ~150–200，≥5 seed | Table 1, Fig 4, Fig 3 | 5090 | ★★★ |
| **B2 理论验证** | 缓存候选上算实测翻转率 vs bound | Fig 5 | 5090（离线） | ★★ |
| **B3 稳健性（吃显存/时间）** | 更长训练、第二域 RECON、多步 rollout、C2 大 $G$ | 附录稳健性 + FVD | H800 | ★★ |
| **B4 主观图** | 随机 + 运动 case 导出 | Fig 6 | 5090 | ★ |

## 7. 依赖与风险（需先确认/拍板）

- **R1（FVD × 单步）**：FVD 需序列，RT-1 single-step 是单帧预测。对策：要么做短 rollout（≥8 帧，B3）再算 FVD，要么单步上用 FID/特征距离替代、FVD 只在多步设定报。**待定**。
- **R2（floor-校准像素 A3 的定义）**：减 $\phi_{\text{tok}}$ 是逐样本还是用期望？建议 reward 用 $-(\text{LPIPS}-\hat\phi_{\text{tok}})$ 并截断非负，$\hat\phi_{\text{tok}}$ 由 B0 估。**待定**。
- **R3（oracle tokenizer）**：交叉验证用哪个更高保真量化级 / 是否需另训。**待定**。
- **R4（第二域 RECON）**：vid2world 数据与 pipeline 是否就绪，决定 B3 能否按时。**待确认**。
- **R5（运动保真度）**：RAFT 较重；单步设定下"光流"需相邻帧，确认 context+预测帧能算出有意义的 flow。**待确认**。
