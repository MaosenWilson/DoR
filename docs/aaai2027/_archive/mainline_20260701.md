## 0. 转向一句话（导师会议 2026-06-27 定调）

> **重点不是"设计更好的 reward"（那只是必要的信号校准，配菜）；重点是"这么丰富的逐段/逐 token 误差，直接交给 GRPO 会出什么问题，以及怎么改 GRPO 的信用分配"（主菜）。**

论文从 **reward 论文** 重构为 **GRPO 信用分配论文**：
- **主贡献 = Codec-Aware Spatial-Temporal Segmental GRPO（CAST-GRPO）**：把 rollout 级标量优势拆成**空间段（+多步时的时间段）**级优势，按 codec 地板做可靠性阻尼，并与全局优势融合。**单步=空间 segmental GRPO;多步=时空 segmental GRPO。**
- **配菜 = Codec-Calibrated Reward**：floor-cancelled 目标对齐 + 弱动态残差（已证，`pixel_tok_dyn λ=0.10`）。只作简要说明，提供"逐段可靠的 reward 信号"喂给 CAST-GRPO。
- **理论地基（已证，最硬）= 地板 φ_tok + 翻转率 arccos(ρ)/π**：既解释为何要校准 reward，也解释为何优势要按段/按地板阻尼——**同一套机理同时撑起配菜和主菜**。

## 1. 问题驱动叙事（导师要的"两步走，先把问题说透"）

### Step 1（配菜的动机）：reward 本身要先定义好，但它有地板、且多层多粒度
RLVR-World 把预测 token 解码成图像，用 MSE/LPIPS 对真值帧逐候选打分。已证事实：
- decoder 有损 → post-decode reward 带**不可约地板** $\phi_{\text{tok}}=\mathbb{E}[d(\text{decode}(\text{encode}(s')),s')]$（与预测质量无关、度量依赖）。
- 不同层次（图像/特征）、不同 loss（MSE/LPIPS/SSIM）给出**不同误差、不同方差**。
- **必要修法（已证，简述）**：把目标从原始 $s'$ 换成可达目标 $\tilde s'=\text{decode}(\text{encode}(s'))$（`pixel_tok`），再加弱 pre-decode 动态残差（`pixel_tok_dyn`, λ=0.10）。这一步**只是把 reward 校准干净**，不是卖点。

### Step 2（主菜的问题）：把这样丰富的误差**直接交给 GRPO 太粗**
标准 GRPO 对每个 rollout 只有**一个标量优势**，广播到该 rollout 的**全部 320 个 token**。但：
1. **空间局部性**：一帧里真正变化的往往只是局部（机械臂/被操作物），大片背景是静态。一个标量把"抓住运动的区域"和"照抄背景的区域"给**同等信用**——错配。
2. **可靠性非均匀**：不同空间段的 codec 地板不同 → 该段 decoded reward 的**可信度**不同。C2 已证地板会翻转组内排序;那么**地板大的段，其优势就该被降权**。
3. **（多步）时间信用**：H>1 时某一未来步错了，不该把 credit 平摊到所有时间步。

> **关键论证**：我们**已经用实验证否了"在全局标量层面修 GRPO"**（Dr.GRPO、硬过滤组 v3 全变差）。这恰好说明——**问题不在全局标量能修的层面，必须下沉到"段级/token 级"的信用分配**。旧负结果 = 新主贡献的动机。

## 3. 方法概要（reward 简述 + CAST-GRPO 详述）

记号：rollout $i\in\{1..G\}$，未来步 $h\in\{1..H\}$（单步 H=1），空间段 $k\in\{1..K\}$。帧 token 网格 **16×20=320**，K 由网格整切（2×2→K=4，块 8×10;4×4→K=16，块 4×5）。

### 3.1 配菜：段级 reward（简述，完整见 `method(1).md §3`）
- 段级 codec 地板 $b_{h,k}=d(\tilde x_{t+h,k}, x_{t+h,k})$。
- floor-cancelled decoded reward $r^{\text{dec}}_{i,h,k}=-[d(\hat x_{i,t+h,k},x_{t+h,k})-b_{h,k}]_+$。
- decoder-前 token reward $r^{\text{tok}}_{i,h,k}$（码空间 L2）。
- 可靠性融合 $q_{h,k}=\exp(-\alpha\tilde b_{h,k})$，$r_{i,h,k}=(1-q_{h,k})r^{\text{tok}}_{i,h,k}+q_{h,k}r^{\text{dec}}_{i,h,k}$。
- （已证的单帧版就是 `pixel_tok_dyn`;段级是它的空间展开。）

### 3.2 主菜：CAST-GRPO（完整见 `method(1).md §4`）
- **段级组内优势 + codec 阻尼**：
$$A^{\text{seg}}_{i,h,k}=\frac{r_{i,h,k}-\mu_{h,k}}{\sigma_{h,k}+\gamma\,\tilde b_{h,k}+\epsilon}.$$
  地板大的段 → 分母大 → 优势被阻尼（正是 C2 的"低可信度降权"）。
- **全局优势**（保全局一致性）$A^{\text{global}}_i=\frac{R_i-\mu_R}{\sigma_R+\epsilon}$，$R_i=\sum_h\omega_h\frac1K\sum_k r_{i,h,k}$。
- **融合**$\hat A_{i,h,k}=\lambda A^{\text{seg}}_{i,h,k}+(1-\lambda)A^{\text{global}}_i$。
- **token 映射**：每个生成 token 按其空间段 $k(t)$（多步再按时间步 $h(t)$）取 $\hat A_{i,h(t),k(t)}$，进标准 GRPO/PPO clipped loss。
- **退化保证**：$\lambda=0$ 或 $K=1,H=1$ → 回到原始 GRPO（bit 级对齐，`--adv_estimator=grpo` 不变）。
- **可选筛选/加权**（导师"先过滤再加权"）：段级可靠性权重（rank-reliability，见 `pipeline_story §7`）作用在 $\hat A$ 上，软加权不丢数据。

## 5. 诚实状态 + 命门（审稿人视角）

**已证（可直接写）**：地板存在且度量依赖;翻转率 $\arccos(\rho)/\pi$（5 seed 吻合）;floor-cancel 收益 ∝ 地板（pixel_tok vs mse_tok）;pixel_tok_dyn(λ=0.10) 动态↑保真不掉;RL≫base;全局 GRPO 改造（Dr.GRPO/硬过滤）失败。

**提出待验（主贡献，风险所在）**：
1. **段级 GRPO 会不会像全局改造一样洗没?** —— 这是最大风险。缓解:它是**不同轴**（局部信用 vs 全局归一），且有 C2 机理支撑;但**必须先单步 pilot 证明**，不能纸上谈兵。
2. **段级归一化方差变大**（每段有效样本仍是 G，但信号更弱）→ 靠 λ 融合全局优势 + `unbiased=False` + eps 稳住。
3. **段级 LPIPS 在小 patch 不稳** → 用整帧 LPIPS broadcast 到段（method(1) §8.2），段级只对 MSE/code 做逐 patch。
4. **token↔空间段映射**：单步在 `dor` 是干净的 16×20 网格（**已确认，绿灯**）;多步 verl 的 token 布局需另确认（implementation_plan §5.1 已标红）。
5. **多步是另一套 infra**（verl，重）;单步不成功不要上多步。
6. **"reward 是配菜"要真配得上**:段级 reward 必须先证明比单帧标量 reward 提供更有区分度的段级信号，否则 CAST-GRPO 无米下锅。

**判据（绿灯才写进主表）**：单步空间 segmental GRPO 在**留出独立指标**（flow + FD-DINOv2/KID/PRDC）上、**配对多 seed（≥5，理想 8）**稳超"同 reward 的原始全局 GRPO";退化实验（λ=0 或 K=1）确认回到 baseline。
