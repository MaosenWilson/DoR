# GSPO 本地化：用 sequence-level ratio 替换原始 GRPO（2026-07-05）

> 目的：REAL rank-label 5 seed 判负后，尝试更正统的 GRPO 替换项：Group Sequence Policy Optimization (GSPO)。本页记录论文/代码来源、公式、本项目本地化方式和 pilot 指令。

## 1. 原始资料

已下载论文：

- `docs/aaai2027/papers/GSPO_group_sequence_policy_optimization_2507.18071.pdf`
- 标题：*Group Sequence Policy Optimization*
- arXiv：2507.18071
- 作者：Qwen Team / Alibaba Inc.

参考代码：

- REAL 仓库内 `external/REAL/verl/verl/trainer/ppo/core_algos.py::compute_policy_loss_gspo`
- 独立参考实现：`external/GSPO-DeepSeek-R1-Distill-Qwen-1.5B/`

核心思想：GRPO/PPO 常用 token-level importance ratio，但 reward 是 sequence-level outcome。GSPO 将 importance ratio 改成序列级：

$$
s_i(\theta)
=
\exp\left(
\frac{1}{|o_i|}
\sum_{t=1}^{|o_i|}
\log
\frac{
\pi_\theta(o_{i,t}\mid q,o_{i,<t})
}{
\pi_{\theta_{\mathrm{old}}}(o_{i,t}\mid q,o_{i,<t})
}
\right).
$$

然后用 sequence-level clipping：

$$
\mathcal L_{\mathrm{GSPO}}
=
-\frac{1}{G}\sum_i
\min\left(
s_i(\theta)A_i,\ 
\mathrm{clip}(s_i(\theta),1-\epsilon_{\mathrm{low}},1+\epsilon_{\mathrm{high}})A_i
\right).
$$

## 2. 为什么适合本项目

我们的视频世界模型每个候选输出是一整段视觉 token 序列（一帧 320 token），而 reward 也是对整帧候选图像/码空间动态的 outcome-level 评价。标准 GRPO 把一个 rollout-level advantage 乘到整个 token logprob sum：

$$
\mathcal L_{\mathrm{GRPO}}=-A_i\sum_t \log\pi_\theta(o_{i,t}\mid q,o_{i,<t}).
$$

GSPO 更匹配这个结构：reward 和 ratio 都在 sequence level，对 tokenized video prediction 是自然替换。

当前固定 reward 仍然使用已验证的：

$$
R_i^{\mathrm{RC}}
=
z_G\big(-\mathrm{LPIPS}(\hat{s}'_i,\tilde{s}')\big)
+
0.10\,z_G(R_i^{\mathrm{dyn}}),
\qquad
\tilde{s}'=D(E(s')).
$$

优势仍为：

$$
A_i=\frac{R_i-\mu_R}{\sigma_R+\epsilon}.
$$

## 3. 实现边界(v2 修订,2026-07-05——v1 有数学死穴,已修)

**v1 的死穴(审查发现,必须记录)**:`dor` 是定长(每帧恒 320 token)、单次 on-policy 更新的 harness。在 $\log\pi_{\text{old}}=\text{stopgrad}(\log\pi_\theta)$ 下:
1. ratio 前向值**恒等于 1** ⇒ clip 区间 $[1-3\text{e-}4,\,1+4\text{e-}4]$ **永不触发**,两个 clip 超参是死代码;
2. 梯度 $=-A_i\cdot\frac1L\sum_t\nabla\log p_t$——GSPO 的长度归一 $1/L$ 只在**变长序列**才改变梯度几何,定长 320 下它就是**常数 1/320**。所以 v1 ≡ vanilla GRPO 把学习率除以 320,"验证 sequence-level objective 是否更稳"的说法**不成立**——它测的只是超小学习率。

**v2 修法(已实现)**:
1. `gspo_loss` 内部乘 $L$ 做定长尺度对齐 ⇒ on-policy 首轮与 vanilla GRPO **梯度逐项相同**(同 lr 可比);
2. 新增 `--ppo_epochs`(≥2 才有 GSPO 内容):同批 rollout 缓存生成时的 $\log\pi_{\text{old}}$,首轮更新后**权重已变**,第 2+ 轮的序列级 ratio 偏离 1、clip 真实生效——这才是 GSPO 论文的实际工作场景(off-policy 重用 + 序列级 clip 稳住更新);
3. 每 eval 周期打印 `ratio_mean/clipfrac`,冒烟时必须看到它们**动起来**(否则等于没测 GSPO)。

**定位(与 C3 的关系,写作时必须讲清)**:C3 证伪的是 advantage/credit **重分配**(污染排序上重排信用无效);GSPO 属于**优化几何/off-policy 稳健**轴,是干预阶梯的**第 4 级**,不与 C3 冲突、也不是主方法翻案。它的实验角色:①若无效 → 阶梯又添一级,强化"有效干预点在 reward 构造";②若有效 → 作为"重用 rollout 提升样本效率"的工程改进写辅助段,主线不变。

**预注册混淆变量**:gspo(E=2)vs vanilla(E=1)混淆了"多一轮更新"与"GSPO objective"。若 gspo 胜出,**必须**补跑 naive-E=2 对照(同样重用 rollout 但无 ratio/clip,即直接再做一次 logp-sum 更新)才能归因;不胜则无需。

## 5. Pilot 指令(v2:必须带同 sweep 基线臂 + ppo_epochs≥2)

smoke(核心验证点:`[gspo] ... ratio_mean` 偏离 1、`clipfrac`>0——否则 GSPO 没被真正测到):

```bash
cd /root/autodl-tmp/vote2world; P=/root/miniconda3/bin/python
$P scripts/train_grpo.py \
  --adv_estimator gspo --ppo_epochs 2 \
  --rewards pixel_tok_dyn --modes gt_only --seeds 0 \
  --steps 3 --K 8 --eval_every 1 \
  --dyn_lambda 0.10 --dyn_gamma 0.25 \
  --out_dir outputs/_gspo_smoke
```

pilot(**同一 sweep 内两臂配对**,5 seed;vanilla 基线不得引用其他批次):

```bash
# 臂1: vanilla 基线(同 sweep)
$P scripts/train_grpo.py --rewards pixel_tok_dyn --modes gt_only --seeds 0,1,2,3,4 \
  --steps 150 --K 16 --eval_every 10 --dyn_lambda 0.10 --dyn_gamma 0.25 \
  --deterministic --out_dir outputs/gspo_pilot_vanilla
# 臂2: gspo(E=2)
$P scripts/train_grpo.py --adv_estimator gspo --ppo_epochs 2 \
  --rewards pixel_tok_dyn --modes gt_only --seeds 0,1,2,3,4 \
  --steps 150 --K 16 --eval_every 10 --dyn_lambda 0.10 --dyn_gamma 0.25 \
  --gspo_clip_low 0.0003 --gspo_clip_high 0.0004 \
  --deterministic --out_dir outputs/gspo_pilot_e2
```

## 6. 判据(赛前锁定)

同 seed 配对(gspo E=2 − vanilla),配对 t:

1. **绿**:flow/dmot 显著提升(t≳2.13, 4/5+)且保真不显著掉 → **先补 naive-E=2 对照**归因,归因成立才写进论文(作为工程辅助段,非主贡献)。
2. **平/负**:干预阶梯第 4 级(优化几何层)添一级负结果,进 C3 素材,一句话+附录。
3. 任何情况下,`pixel_tok_dyn + vanilla GRPO` 保持主方法不变。
