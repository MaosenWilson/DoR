## 1. 方法设计（现在的样子）

**CAST-GRPO = Codec-Aware Spatial-Temporal Segmental GRPO**。核心思想：标准 GRPO 给每个候选一个标量优势，广播到全部预测 token；一帧图像里真正在变化的往往只是局部区域，把优势下沉到**空间段级**（单步）或**时空段级**（多步），理论上能更精细地做 credit assignment。

**记号**：转移 $q=(s,a)$，策略采样 $G=16$ 个候选未来帧 token 序列。单步 tokenizer 把每帧编成 $16\times20=320$ 个 token（干净空间网格，可整除切成 $K=4$ 或 $K=16$ 段）。

**段级优势公式（骨架，稳定不变，未被证伪）**：
$$A^{\text{seg}}_{i,k}=\frac{r_{i,k}-\mu_k}{\sigma_k+\gamma\tilde b_k+\epsilon},\qquad \hat A_i=\lambda A^{\text{seg}}_i+(1-\lambda)A^{\text{global}}_i$$
$\lambda=0$ 或 $K=1$ 应退化回标准全局 GRPO（这个退化性质本身还没有被严格验证过，见 §6）。

**这套骨架本身没有被证伪**——被反复证伪的是"喂给这套骨架的 reward 该怎么算"（见 §5）。

## 2. 已证明的结论（不需要重新验证）

### 2.1 地基机理（B）

| 结论 | 数字 | 协议 |
|---|---|---|
| Tokenizer 解码引入不可约"重建地板"，与预测质量无关，度量依赖（感知度量地板大、MSE 小） | 定性+跨度量特征表 | eval-only 探针 |
| **翻转率理论**：地板腐蚀 GRPO 组内排序，翻转概率 = $\arccos(\rho)/\pi$（$\rho$=含噪奖励与干净参照的相关性） | 实测翻转率 **0.185** vs 理论预测 **0.186**，5 seed 一致；弱信号窗口翻转率更高 | 无参照（用 code 空间当干净参照）|

**这是全文最硬的结果**，不依赖任何后续设计是否成功。

### 2.2 Reward 校准（C，已冻结为 `pixel_tok_dyn`）

| 结论 | 数字 | 备注 |
|---|---|---|
| floor-cancellation 主效应：把比较目标从原始 GT 换成"tokenizer 可达目标" | pixel→pixel_tok：flow **+14%**，5/5 seed 一致 | C3b |
| mse_tok 负对照：增益应正比于该度量的地板大小 | LPIPS 地板大→cancel 后 5/5 seed 一致；MSE 地板小→cancel 后仅 3/5、方向不稳 | 支持"floor-cancel 有效是因为地板"而非任意换目标 |
| 动态残差权重扫描，选定 Pareto 点 | λ_dyn=0.05：flow 0.2488±0.0163（太弱）；**λ_dyn=0.10：flow 0.2877±0.0257（最佳）**；λ_dyn=0.25：flow 0.2822±0.0240 但保真明显变差 | 最终选定 `pixel_tok_dyn`(λ_dyn=0.10, γ_dyn=0.25) |

**`pixel_tok_dyn` 的公式**：$R_i = z_G(-\text{LPIPS}(\hat s'_i,\tilde s')) + 0.10 \cdot z_G(R^{\text{dyn}}_i)$，$R^{\text{dyn}}=\cos(\Delta z_i,\Delta z')-0.25|\log(\|\Delta z_i\|/\|\Delta z'\|)|$（码空间方向+幅度残差）。

**这一层不再扩展**（导师明确要求：reward 是 GRPO 的输入信号，不是本文的核心贡献，不要继续加新指标/加权/搜索）。

## 3. 已证伪的结论（重要：避免重复踩坑）

| # | 尝试 | 结果 | 根因 |
|---|---|---|---|
| F1 | "code（解码前）reward 普遍比 pixel 更好" | ❌ 证否 | flow 优势在更全的 5-seed 扫描里不复现；分布/特征面板（FD-DINOv2/KID/PRDC）各臂几乎无差；全臂在分布层面出现 recall 坍塌 |
| F2 | 全局标量层面"改良 GRPO"（Dr.GRPO 去偏、硬过滤地板主导组 v3） | ❌ 证否 | Dr.GRPO 中性无益；硬过滤组反而全面变差（丢数据 + 破坏 std 对地板组的天然降权）|
| F3 | 段级信用分配用在**信息量弱的裸 code reward** 上 | ❌ 证否（负对照） | λ 越大动态指标越差；佐证"段级信用需要信息丰富的 reward 才有意义" |
| F4 | 数据集级、离线算一次的全局 Wiener 权重融合多指标 reward（`dorw` 臂） | ❌ 证否（历史结果） | 塌缩成纯 code：LPIPS/MSE 的**数据集平均**地板相对**数据集平均**信号太大，权重被压到 ~0，等于放弃了 decoded reward 里所有信息 |
| F5 | `codec_fused` v1：可靠性权重 $q=\exp(-\alpha\tilde b)$（地板越大权重越小，启发式）| ❌ 废弃 | 拍脑袋公式，$\alpha$ 从未调过，无推导依据 |
| F6 | `codec_fused` v2：可靠性权重用方差分解 $q=\hat\sigma^{*2}/(\hat\sigma^{*2}+b^2)$，把地板**均值** $b$ 当噪声标准差 | ❌ 证伪 | 真实数据实测 **q≡0**（K=1/4/16 全部塌缩，98-100% 段 q<0.01）。根因：地板的均值部分是组内常数，GRPO 组归一化会自动减掉，**不腐蚀排序**；真正腐蚀排序的是**候选间的地板波动** $\sigma_\eta\ll b$（这正是 §2.1 翻转率理论自己的论点）。用 $b^2$ 当噪声方差高估了几十倍。**这个根因同时回溯解释了 F4（`dorw`）的塌缩**——同一病根，两次发作 |
| F7 | `codec_fused` v3：可靠性权重改用相关性 $q=[\max(0,\rho)]^2$（$\rho$=组内 corr(decoded reward, code reward)，即 §2.1 翻转率理论里同一个 $\rho$ 的段级在线版），但把 decoded reward 的比较目标错设为**原始 GT**（丢失了 C3b 的目标对齐技巧） | ❌ 未过绿灯 | q 的真实数据分布这次健康了（K=1 q≈0.85、K=4 q≈0.56、K=16 q≈0.41，段间有真实分化，不再塌缩），**证明可靠性权重机制本身工作正常**；但整体训练效果差：λ=0.7 vs λ=0 配对，flow 几乎不变（t=0.47，噪声内），dmot 弱（t=1.57），**保真三项一致显著变差**（LPIPS t=−3.99, PSNR t=−4.77, SSIM t=−4.22）|
| F8 | `codec_fused` v3.1：在 v3 基础上只改一处——decoded reward 比较目标换回"可达目标"（恢复 C3b 技巧），其余不变 | ❌ 仍未过绿灯 | 保真惩罚减半（PSNR t 从 −4.77 降到 −2.21），说明目标对齐确实起作用、诊断方向对了，**但幅度不够**：flow +0.006（3/5，t=0.50，噪声）、dmot +0.014（4/5，t=2.47，勉强）、保真三项仍显著变差（LPIPS t=−2.02, PSNR t=−2.21, SSIM t=−1.92）|

**F5→F8 是同一条主线的四次迭代（v1→v2→v3→v3.1），核心教训**：
1. 可靠性加权的"噪声量"必须锚定在**候选间波动**，不能用**地板均值**代替（F6 的教训，一次性解决了 F6 和历史遗留的 F4）。
2. 但即便噪声量估计对了（v3/v3.1 的 q 分布健康），**去掉"目标对齐"这个已证技巧会让整体效果变差**（v3 vs v3.1 的保真差距对比证明了这一点）；即使补回目标对齐，**幅度仍不够翻盘**——说明问题不只在 q 的公式本身。

## 5. 当前悬而未决的核心问题（§4 配置的可信度存疑）

### 问题 A13：段池化基线本身可能在损伤 reward，段级信用只是"收复失地"

跨协议对比（**注意：以下两行不是同一 sweep 配对跑出来的，存在协议差异风险，见下）**：

| 配置 | flow | 说明 |
|---|---|---|
| **真全局** `pixel_tok_dyn`（标准 `--adv_estimator grpo`，整帧算 reward，λ_dyn=0.10，n=5，历史数据） | **0.2871±0.0153** | 全表所有配置里最高 |
| segpd λ=0.0（同一个 reward 配方，但**先按段池化、再取平均**，n=5） | **0.2142±00288** | 明显更低 |
| segpd λ=0.7（段级信用打开） | 0.2680±0.0286 | 比 λ=0 好很多，但还没追上真全局 |

**读法**：把 reward 拆成段、每段算一次再池化，这个动作**本身**似乎就比整帧直接算一次 reward 要差（0.287→0.214，掉了 25%）。段级信用打开后爬回 0.268，**看起来更像是"弥补段池化造成的损失"，而不是"在一个健康的基线上锦上添花"**。

**这两行数字不是同一批次配对跑出来的**（真全局那行是更早期的历史数据，协议参数虽然记录相同但没有做过严格的同 sweep 配对验证，跨批次存在已知的 GPU 非确定性噪声，量级可达 ±0.03-0.08，见 §7）。所以目前**不能排除这个差距部分是噪声**，但差距（0.287 vs 0.214，Δ=0.073）明显大于历史观测到的噪声量级上限（±0.08 是极端值，通常更小），值得认真对待。

**这是 Table 1 能否成立的最后一道关卡**：如果段级信用只是打平真全局、没有真正超过，那"段级信用分配"这个核心卖点就站不住；如果它能显著超过真全局，主贡献成立。

**需要的实验**（尚未执行，仅供参考，不代表下一步一定这么做——这正是要先分析清楚的部分）：同一个 sweep 里配对跑「真全局 pixel_tok_dyn」vs「segpd λ=0/0.7/1.0」，同 seed、同协议、更大样本量（比如 8 seed），直接配对比较，而不是引用历史数据。

### 其他待厘清的问题

1. **§1 提到的退化性质（λ=0 或 K=1 应该退化回真全局）从未被严格验证过**——这可能和问题 A13 是同一件事的两个侧面：如果连"退化" configuration 都对不上真全局，那"段池化"这个实现细节本身可能有问题（不只是"信息量损失"这么简单，可能是具体实现在算法层面就不等价于全局）。
2. **codec_fused（q 加权）这条线要不要继续救**：v3.1 已经把保真惩罚减半，如果继续调（比如换成 §2 提过的逆方差形式 $w=\sigma_\star^2/(\sigma_\star^2+\sigma_\eta^2)$、或者调整 z-score 细节、或者重新检查 dyn 项的融入方式）是否有希望，还是应该彻底放弃、只把它当消融写进论文？
3. **段级 reward 的池化方式本身**（当前是对每段做 RMS 后 z-score）是否是信息损失的来源？有没有更保信息的池化方式？
4. **γ（codec-aware 分母阻尼）的独立贡献从未被单独消融过**——目前 segpd 用的 γ=0.2 是沿用的旧值，没验证它本身是加分还是无关变量。

## 7. 方法论注意事项（分析时必须考虑）

1. **GPU 非确定性**：同一 seed、同一配置，两次独立运行的 flow 可以相差 ±0.03~0.08（根因是 flash attention backward 的非确定性，`--deterministic` 只 pin 了 cuBLAS/cuDNN，不能做到逐位可复现）。**跨 sweep 批次比较（比如 §5 A13 引用的历史全局数据）存在这个噪声风险**，同 sweep 内的配对比较更可信。
2. **z-score 惯例**：不同奖励分量（码空间 RMS vs LPIPS 残差）原始尺度差一个数量级，融合前必须先做段内 z-score，否则尺度而非可靠性会主导融合结果。
3. **地板量的两种用法不能混用**：归一化到均值~1 的地板 $\tilde b$（用于优势分母阻尼 γ 项）和原始尺度的地板 $b$（如果要做方差分解，必须用原始尺度，F6 的教训）是两个不同的量，单位不一致会导致公式行为不对。
4. **`--seg_reward` 的历史版本**：`code`(裸码空间,已证否)、`pixtok`/`pixtok_dyn`(§2.2 配方的段级搬运,即"最佳已知方法")、`codec_fused`(v1→v3.1 四版失败迭代,现已停止)。
