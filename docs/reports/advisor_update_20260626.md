# Mind the Reconstruction Floor：为 tokenized 视频世界模型校准可验证奖励

**日期**：2026-06-26  

> 一句话：我们发现 RLVR 训练视频世界模型时，"可验证奖励"算在**解码后**的像素上，带一层与预测好坏无关的**噪声地板**，它会腐蚀强化学习真正依赖的"组内排序"；我们提出把奖励改到**解码前的码空间**来根治，并已用实验验证机理与有效性。

---

## 〇、术语速览

**A. 模型与生成**
- **世界模型 (world model)**：会"想象未来"的模型——输入当前几帧画面 + 要执行的动作，输出预测的下一帧。相当于学会了环境动态的模拟器。
- **tokenizer / token / 量化 (quantization)**：模型不直接处理像素，而是先用一个"压缩器"把每帧图像转成一串**离散整数编码(token)**，相当于把图像翻译成一套"视觉词汇"。`encode`=图像→token，`decode`=token→图像。量化=把连续特征归到有限个"码字"上。**这个压缩是有损的**——decode 回来的图和原图必有差别。
- **自回归生成 (autoregressive)**：像写句子一样，一个 token 接一个 token 地把下一帧"写"出来。

**B. 强化学习部分**
- **强化学习 / 奖励 (reward)**：让模型对同一输入生成多个候选，给"好"的候选高分（奖励），把模型推向高分行为。
- **可验证奖励 (verifiable reward, RLVR)**：奖励不是另一个模型主观打的，而是用**真值**直接算的客观距离（预测帧和真实下一帧有多接近），故"可验证"。RLVR-World 就是这套范式的代表工作。
- **GRPO**：本文用的 RL 算法。它对同一输入采样**一组 K 个**候选，**只看这组候选彼此谁好谁坏的排序**（做"组内归一化"：减组内均值、除组内标准差），据此更新模型。**关键：它只消费组内相对排序，不在乎绝对分值。**
- **组内排序**：一组 K 个候选按奖励从高到低的次序——GRPO 学习完全依赖它。

**C. 图像/视频质量指标（都是"两帧有多像"）**
- **MSE**（均方误差，↓越好）、**PSNR**（峰值信噪比，↑越好）：逐像素差异。
- **SSIM**（结构相似度，↑越好）：比较亮度/对比度/结构。
- **LPIPS**（感知距离，↓越好）：用神经网络特征算，最接近人眼判断。（有 alex/vgg 两种骨干，数值不同，须同骨干才能比；本文用 vgg，和 RLVR-World 对齐。）
- **光流 / flow / RAFT**（↑越好）：光流=画面中每个点"往哪移动多少"的向量场，RAFT 是估计光流的网络。**flow 一致性**=预测帧的运动方向是否和真实运动一致——**直接衡量"动态"对不对**（正中本文主题）。
- **FID / FVD（分布级真实度）**：不逐张配对，而是把一批生成图/视频和一批真实图各自送进预训练网络(Inception/I3D)提特征，比两堆特征的整体分布差多远——**越小越像真实的那批**。一句话：**LPIPS 问"准不准"，FID 问"真不真"**（能抓到逐张指标抓不到的"整体偏糊/纹理假"）。FVD 是其视频版。**注：RLVR-World 的视频评测并未上报 FID/FVD。**

**D. 本文核心概念**
- **奖励噪声地板 φ_tok (reward noise floor)**：因为 decode 有损，**任何**预测帧解码后都自带一层固定误差，即便 token 完全正确也消不掉——就像信号处理里"低于噪声地板的信号无法被检测"。**怎么衡量**：拿真实帧 s'，做一次理想往返 `decode(encode(s'))`，量它和 s' 的距离 d，对大量帧取平均，即 $\phi_{\text{tok}}=\mathbb{E}[\,d(\text{decode}(\text{encode}(s')),\,s')\,]$。**注意这里 d 可以是任意度量（MSE / SSIM / LPIPS / FID…）——地板是"解码有损"的直接后果，对每一种后解码度量都存在（存在性普遍）；不过它是否大到"淹没"动态信号则随度量而变：在 LPIPS/SSIM 上地板 ≥ 帧间信号（最严重），在 L2(MSE/PSNR) 上地板更小（见进展4）。** vgg 下实测 LPIPS 地板 ≈ 0.112 > 帧间信号 ≈ 0.082。
- **FSQ 码空间**：FSQ 是该 tokenizer 的量化方式；"码空间"=token 对应的量化向量，在**解码前**即可取得。在码空间算奖励**不经过有损解码器，故无地板**——这是本文方法 DoR 的落点。
- **排序翻转率 (rank-flip rate)**：地板是噪声，可能把两个候选的奖励高低**颠倒**。翻转率=一组里有多少对候选的排序被噪声颠倒。理论上界 $\arccos(\rho)/\pi$（高斯噪声下），ρ=干净奖励与带噪奖励的相关系数。

**E. 统计与工程术语**
- **seed（随机种子）/ std（标准差）**：同一实验换 5 个种子各跑一次，看结果稳不稳；std 是波动幅度。
- **发散 (divergence)**：训练失控，结果变得异常差。
- **配对检验 / t 值 / 显著性**：统计上判断两组结果差异是否真实（非偶然）；经验上 |t|>2.78（5 seed）≈ p<0.05 才算"显著"。
- **checkpoint**：训练完的模型存盘，以后可直接加载评测，无需重训。

## 一、研究目标（我们想解决什么）

### 1. 出发点：RLVR-World
- 视频世界模型：给定历史帧 + 动作，预测下一帧。当前代表性训练范式是 **RLVR-World**（THU, arXiv:2505.13934）——用 GRPO 做强化学习，奖励是"预测帧 vs 真值帧"的可验证距离（**−LPIPS**）。
- 它的范式被默认为"可验证 = 可信"。

### 2. 我们发现的问题
- 世界模型先把图像**量化成离散 token**再自回归生成；但奖励是把 token **解码回像素之后**才算的。
- **解码是有损的**：哪怕预测的 token 完全正确，`decode(encode(·))` 往返也有不可约误差。我们把它定义为**奖励噪声地板 φ_tok**（见术语 D）。
- **这不是 LPIPS 一个指标的毛病，而是"在解码后算奖励"这件事的毛病**：换成 MSE、SSIM、还是多分量融合，地板照样存在。普遍性由两条佐证——① φ_tok 对任意度量 d 都 >0（decode 有损的直接推论）；② 实验里 A1(MSE)/A2(SSIM)/A4(multi) 这些不同的后解码奖励**都没能胜过 code**。
- 以 LPIPS（RLVR-World 的奖励度量）为例，实测地板 ≈ 0.112 > 帧间真实运动 ≈ 0.082 —— **在这个度量上地板淹没了动态信号**（进展4 给出全度量对比：感知/结构度量都被淹没，L2 则有余量）。

### 3. 为什么这会坏事（核心洞察）
- GRPO 只消费一组候选的**组内相对排序**（见术语 B）。
- 把奖励写成 R = 真实质量信号 + 地板噪声。当**动态信号的组内波动 σ²⋆ 小于地板噪声波动 σ²η** 时，组内排序被噪声**翻转** → 梯度方向系统性出错 → **策略在学噪声，而非动态。**

### 4. 我们的目标与假设
> **目标**：证明这层奖励噪声地板真实存在且有害，讲清它"怎么害"的机理，并给出根治的奖励设计。
>
> **假设（方法 DoR）**：奖励不该在解码后的像素算，而应在 tokenizer **解码前**的空间算。**FSQ 码空间**（解码前、≈零地板、在 tokenizer 内部、零额外开销）是最优落点。

**贡献是一条线**：问题（地板存在）→ 机理（地板腐蚀组内排序）→ 解法（换到码空间）。

## 二、实验设计（我们怎么验证）

### 1. 统一控制（保证公平）
所有奖励**同一个 pre-RL 基座** `thuml/rt1-world-model-single-step-base`，同 GRPO、同候选数 K=16、同 150 步、**每个设定 5 个随机种子**，**只更换"奖励在哪个空间算"**这一个变量。

### 2. 奖励空间扫描：从"解码后"逐步移到"解码前"
| 臂 | 奖励度量 | 空间 | 角色（它要回答的问题） |
|---|---|---|---|
| A0 | −LPIPS | 像素（解码后） | **RLVR-World 基线** |
| A1 | −MSE | 像素 | 换个像素指标行不行 |
| A2 | SSIM | 像素（结构） | 换个结构指标行不行 |
| A3 | 减地板的 −LPIPS（常数 / 逐候选） | 像素 | **"简单把地板减掉"够不够** |
| A4 | 多分量融合(ToolRL 式) | 像素 | **"堆更复杂的像素奖励"够不够** |
| A5 | −RMS(解码前连续特征) | 解码前 | 量化到底帮还是伤 |
| A6 | **−RMS(FSQ 码) = DoR** | 解码前 | **本文方法** |

（RMS=均方根距离，即两向量逐元素差的平方平均再开根，越小越近。）

### 3. 评测口径：与 RLVR-World 一一对齐（不只看 LPIPS）
- **直接复用 RLVR-World 官方评测器的全套指标**：**MAE / MSE / PSNR / SSIM / LPIPS(vgg)**，逐一对标其论文（这是早期只用 LPIPS-alex 的修正——既换了骨干为 vgg，也补齐了五项指标）。
- 另加**动态指标 flow**（RAFT 光流一致性）——直接量运动预测对不对，正中"Dynamics"。
- 同源指标（码空间自身距离，与 A6 同空间）只作内部参考，**不进主结论**（避免"自己考自己"）。
- 这样无论用哪一项指标读，结论都不依赖"我们恰好挑了 LPIPS"。
- **关于 FID/FVD**：RLVR-World 视频 RT-1 评测**未上报** FID/FVD（代码有此能力但未启用），我们已对齐其**全部上报指标**；FID 是可选的额外分布级指标，已存 checkpoint 可随时纯评测补、无需重训。

### 4. 机理验证（独立实验，不依赖任何外部"真值参照"）
以解码前 code 的组内排序为"干净"参照，测解码后像素奖励把它**翻转的比例**，对照理论上界 $\arccos(\rho)/\pi$，看实测是否吻合、是否在弱信号区更严重。

### 5. 范围说明
聚焦 **RT-1 单步预测**（与 RLVR-World 的单步设定对齐；多步是其另一独立工作，本阶段不涉及）。

## 三、现在的进展（已完成 + 结果）

### 进展 1：奖励空间扫描主表（A0–A6 × 5 seed，已跑满 40/40）
held-out 独立指标，5 seed 均值±std（`*`=该臂自身优化的指标，有"主场优势"，看 DoR 应看非 `*` 列）：

| 奖励臂 | LPIPS-vgg ↓ | PSNR ↑ | SSIM ↑ | flow(动态) ↑ | 稳定性 |
|---|---|---|---|---|---|
| **A6 code (DoR)** | **0.140 ± 0.002** | 25.26 ± 0.14 | 0.800 ± 0.003 | **0.260 ± 0.018** | **5/5 全稳** ✅ |
| A2 SSIM | 0.140 ± 0.002 | 25.40 ± 0.06 | 0.804* ± 0.001 | 0.207 ± 0.038 | 重建好 / **动态最差** ⚠️ |
| A0 pixel (RLVR-World) | 0.160* ± 0.036 | 24.68 ± 1.04 | 0.781 ± 0.035 | 0.262 ± 0.028 | **1/5 seed 发散** ⚠️ |
| A1 −MSE | 0.148 ± 0.003 | 24.94 ± 0.12 | 0.793 ± 0.003 | 0.236 ± 0.021 | 动态偏低 |
| A3 floor(常数) | 0.143* ± 0.001 | 25.14 ± 0.05 | 0.797 ± 0.001 | 0.263 ± 0.028 | ≈ A0（排序不变）|
| A3 floor(逐候选) | 0.147* ± 0.003 | 24.79 ± 0.23 | 0.790 ± 0.005 | 0.225 ± 0.038 | 减地板更差 |
| A4 multi(ToolRL) | 0.143* ± 0.002 | 25.19 ± 0.13 | 0.799 ± 0.002 | 0.253 ± 0.035 | |
| A5 phi(解码前连续) | 0.143 ± 0.003 | 25.15 ± 0.18 | 0.796 ± 0.005 | 0.250 ± 0.041 | 稳，略逊 code |

**三点结论：**
- **A6 code (DoR) 是唯一"重建(LPIPS/PSNR/SSIM) + 动态(flow)"双轴都进第一档、且 5 seed 全稳的奖励。**
- **A2 SSIM：重建追平 code，但动态(flow)最差** → 结构奖励**牺牲动态换重建**，正是"重建好≠动态对"的反例。
- **A0 忠实 RLVR-World 基线不稳**：4/5 seed≈0.143，但 1 个发散到 0.225；**减地板(A3)也救不了**（常数版对排序无影响≈A0，逐候选版更差）。

### 进展 2：机理被精确验证（无参照，5 seed）
- 实测组内**翻转率**与理论上界 $\arccos(\rho)/\pi$ **5 seed 全部吻合到 ~0.002**（pixel：实测 0.185 / 上界 0.186）。
- **弱信号窗口翻转率高达 ~30%**（强信号窗口仅 ~10%），`相关系数(翻转率, 信号强度) = −0.79`。
- → "地板腐蚀组内排序、信号越弱越严重"被数据坐实。这是全文最硬的一块证据（不依赖任何外部参照）。

### 进展 3：内生消融（量化的作用）
A5（解码前**连续**特征）与 A6（解码前**量化**=code）**都稳、都进第一档**（验证"解码前"是关键，非偶然）；其中 code 在 LPIPS/flow 上**略优、且方差更小** → 量化不损奖励信号、反而更稳。

### 进展 4：跨度量地板（无训练，eval-only，~20 秒）
对每个真值帧 s'，比较两个量在五个度量下的大小：**地板** = `d(decode(encode(s')), s')`（tokenizer 往返误差）vs **信号** = `d(prev, s')`（帧间真实变化）：

| 度量 | 地板(往返) | 信号(帧间) | 谁大 |
|---|---|---|---|
| LPIPS | 0.112 ± 0.019 | 0.082 ± 0.043 | **地板 ≥ 信号** |
| SSIM | 0.841 ± 0.038 | 0.902 ± 0.052 | **地板 ≥ 信号**（往返比"复制上一帧"还不像 GT）|
| MAE | 0.0246 ± 0.005 | 0.0226 ± 0.013 | 地板 ≈ 信号 |
| MSE | 0.0019 ± 0.001 | 0.0034 ± 0.004 | 地板 < 信号 |
| PSNR | 27.40 ± 1.4 | 27.29 ± 5.6 | 地板 < 信号（几乎持平）|

**两点结论（诚实，且把诊断接到训练结果）：**
- **地板在每个度量都存在（>0）；但"淹没信号"只发生在感知/结构度量（LPIPS、SSIM、MAE），不在 L2（MSE、PSNR）。** 因为机械臂运动是局部大像素差，L2 放大它 → 信号盖过地板；而 LPIPS/SSIM 量感知/结构，tokenizer 往返恰丢这部分 → 地板盖过信号。
- **"地板/信号比"反过来精确预测主表的发散**：LPIPS 地板 ≥ 信号 → A0（RLVR-World 的 −LPIPS）最脆弱、1/5 发散；MSE 地板 < 信号 → A1（MSE）有余量、稳定不发散。**RLVR-World 偏偏选了最糟的 LPIPS。** —— 诊断（地板）↔ 结果（发散）形成机制闭环。

## 四、阶段结论

1. **机理坐实**：像素可验证奖励存在噪声地板，腐蚀 GRPO 组内排序，翻转率符合闭式上界（无参照、跨 5 seed）。
2. **方法有效**：DoR 码空间奖励是唯一**可靠（5/5 不发散）+ 重建与动态双优**的奖励；像素-LPIPS 会发散，SSIM 丢动态。
3. **零开销**：码空间在 tokenizer 内部，相对像素奖励无额外网络。

## 五、诚实标注

- 单步设定下，code 对 pixel 的**重建**均值领先，因 pixel 高方差而**配对检验未达显著**（t≈−1.2）；code 的可证优势集中在**可靠性 + 动态 + 机理**。
- 动态指标在单步只注入一次地板、区分度有限；**多步预测预计放大差距**（按单步聚焦暂未做，想请教是否要上）。
- FVD/FID（分布级真实度）尚未加，但已为所有训练**存 checkpoint**，可纯评测补算、无需重训（待落实 I3D 权重）。
